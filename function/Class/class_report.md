# Class 폴더 구조 및 내용 분석 보고서

## 1. 폴더 구조

- class_01.py ~ class_14.py: 클래스와 객체지향 프로그래밍의 다양한 주제를 다루는 예제 파일
- class_review01.py: 클래스 개념 종합 복습 예제

## 2. 주요 파일별 상세 코드 설명

- **class_01.py**: 클래스 변수와 인스턴스 변수의 차이를 실습합니다. 객체별로 name 속성을 다르게 할당할 수 있고, 클래스 변수는 모든 객체가 공유하지만, 인스턴스에서 재할당하면 독립적으로 동작함을 보여줍니다.

- **class_02.py, class_03.py**: 클래스 변수(serial_num)를 활용해 객체마다 고유한 일련번호를 부여합니다. 생성자에서 클래스 변수를 증가시키고, 인스턴스 변수로 복사해 각 객체가 자신의 번호를 갖게 합니다.

- **class_04.py**: 생성자(__init__)에서 인스턴스 변수를 정의하고, 객체 생성 시 자동으로 호출됨을 보여줍니다. 클래스와 인스턴스 변수의 차이, 객체 생성 방법을 실습합니다.

- **class_05.py**: 학생 관리 클래스를 만들고, 인스턴스 변수(name, age)와 메서드(info_student)를 통해 학생 정보를 출력합니다. 리스트를 활용해 여러 학생 객체를 관리할 수 있습니다.

- **class_06.py**: 가장 기본적인 클래스와 객체 생성 예시입니다. 생성자에서 아무 동작도 하지 않지만, 객체를 만들고 인스턴스 변수/메서드를 사용할 수 있음을 보여줍니다.

- **class_08.py**: 학생 클래스 예제. 이름, 국어, 영어, 수학 점수를 인스턴스 변수로 저장하고, 총점(total), 평균(average), 학점(grade) 메서드와 __str__ 메서드를 통해 학생 정보를 출력합니다. 객체지향적 데이터 관리와 메서드 활용의 좋은 예시입니다.

- **class_09.py**: 가위바위보 게임 클래스를 구현합니다. 사용자 입력과 컴퓨터의 랜덤 선택을 받아 승패를 판별하고, 반복적으로 게임을 진행합니다. 입력 검증, 반복문, 클래스 변수/메서드 활용 등 실전적인 객체지향 설계가 돋보입니다.

- **class_10.py**: 숫자 맞추기 게임. 컴퓨터가 랜덤으로 숫자를 정하고, 사용자가 맞출 때까지 입력을 반복합니다. 입력값 검증, 시도 횟수 카운트 등 실전 게임 설계에 필요한 요소를 포함합니다.

- **class_11.py**: isinstance() 함수를 활용해 객체의 타입을 확인하고, 다형성을 지원하는 예시입니다. 리스트에 다양한 객체(Student, Teacher)를 넣고, 타입에 따라 다른 메서드를 호출합니다.

- **class_12.py**: 클래스 콜백 메서드(__eq__, __str__ 등)를 오버라이딩하여 객체 비교, 출력 등 파이썬의 특수 메서드 활용법을 실습합니다.

- **class_13.py**: property, getter, setter, private 변수 사용법을 보여줍니다. _age와 같은 변수에 접근 제어를 하고, @property와 @setter를 통해 안전하게 값을 읽고 쓸 수 있게 합니다.

- **class_14.py**: 상품관리 클래스를 종합적으로 설계합니다. 생성자, property, getter/setter, 입력값 검증 등 객체지향의 다양한 기능을 통합적으로 실습합니다.

- **class_review01.py**: 클래스/인스턴스 변수, 생성자, 인스턴스 생성 및 변수 변경 등 클래스의 핵심 개념을 종합적으로 복습합니다.

## 3. 심화 분석: 주요 예제

### class_08.py (학생 클래스)
- **구조**: 학생의 이름, 국어/영어/수학 점수를 인스턴스 변수로 저장. 총점, 평균, 학점 계산 메서드와 __str__로 정보 출력.
- **객체지향적 설계**: 데이터(점수)와 기능(계산, 출력)을 하나의 객체로 묶어 관리. 확장 시 과목 추가, 학년/반 등 다양한 속성 추가 가능.
- **개선/확장 아이디어**: 과목을 리스트/딕셔너리로 관리, 학생 목록을 관리하는 클래스 추가, 파일 입출력 등.

### class_09.py (가위바위보 게임)
- **구조**: 사용자 입력과 컴퓨터 랜덤 선택, 승패 판정, 반복 게임 진행. 입력 검증 및 예외처리 포함.
- **객체지향적 설계**: 게임의 상태와 로직을 클래스에 캡슐화. 메서드 분리로 유지보수 용이.
- **개선/확장 아이디어**: 점수 집계, 게임 기록 저장, GUI 적용 등.

### class_13.py (getter/setter, property)
- **구조**: private 변수(_age)와 property를 활용해 안전하게 값 접근/수정. @property, @setter의 파이썬적 활용.
- **객체지향적 설계**: 캡슐화와 정보 은닉의 전형적 예시. 외부에서 직접 변수 접근을 막고, setter에서 유효성 검사 가능.
- **개선/확장 아이디어**: setter에서 값의 범위 제한, 삭제자(deleter) 추가 등.

### class_14.py (상품관리 클래스)
- **구조**: 상품명, 가격, 재고를 인스턴스 변수로 관리. 가격에 대한 getter/setter로 유효성 검사.
- **객체지향적 설계**: 속성별 접근 제어, 생성자에서 초기화, property로 안전한 데이터 관리.
- **개선/확장 아이디어**: 재고 관리 메서드 추가, 상품 목록 관리, 할인/이벤트 기능 등.

## 4. 전반적 특징 및 학습 포인트

- 클래스 변수와 인스턴스 변수의 차이, 생성자와 메서드, property, getter/setter 등 객체지향의 핵심 개념을 실습 중심으로 학습
- 게임(가위바위보, 숫자 맞추기) 등 실생활 예제를 통해 클래스 설계와 활용법을 익힘
- 특수 메서드(__str__, __eq__ 등)와 property를 통한 파이썬 객체지향의 고급 기능까지 폭넓게 다룸
- 각 파일은 주석과 함께 개념별로 분리되어 있어 단계별 학습에 적합
